# AI Agent 核心概念与运行机制知识整理

本文档基于 `docs/images/` 下的 7 张知识图片整理而成，系统对比了语言模型（LM）、工作流（Workflow）与智能体（Agent）的区别，并详细拆解了 Agent 的组成结构与核心运行机制。

---

## 一、LM、Workflow 与 Agent 的核心区别 (1.png)

1. **语言模型 (LM - Language Model)**：
   * **特点**：会聊天。
   * **定位**：提供交互界面和基础的语言处理能力。
2. **工作流 (Workflow)**：
   * **特点**：按流程干活。
   * **定位**：由人提前编排好规则，AI 按照既定步骤串行执行。
3. **智能体 (Agent)**：
   * **特点**：自己想办法干活。
   * **定位**：围绕设定目标，自主规划路径、选择工具并解决问题。

> **核心结论**：Agent 的核心不在于“拥有的工具多不多”，而在于“能不能自主规划并完成任务”。

---

## 二、大语言模型 (LM) 的本质与局限 (2.png)

* **本质**：是一个会聊天的大脑，类似于“百科全书”。
* **典型代表**：DeepSeek、GPT 等大语言模型（LLM）。
* **工作模式**：`输入 (Input)` -> `LLM` -> `输出 (Output)`。
  * *例如*：用户问“给我一份会议纪要模板”，模型根据通用知识库输出一个通用的会议纪要模板。
* **面临的问题**：**缺乏个人上下文与实时外部连接**。
  * *例如*：当用户问“我上一次会议是什么时候？”，LM 无法回答，因为它没有接入用户的个人日历、会议记录或本地系统。

---

## 三、工作流 (Workflow) 的运行模式 (3.png)

* **定义**：人提前设计好并固化下来的业务流程（按特定路线走）。
* **任务示例**：“把上一次会议纪要总结后发邮箱”
  * 步骤：① 查日历找到上次会议 -> ② 获取会议记录 -> ③ 调用 LM 做摘要 -> ④ 接入邮箱 -> ⑤ 发送邮件。
* **核心特征**：
  * 步骤可以非常繁多（即使有 100 个节点，它依然是 Workflow，而不是 Agent）。
  * 所有的分叉、跳转和顺序路线全部是由**人类开发/设计者提前写死**的。
  * **AI 只是在既定流程中执行具体的原子节点任务，不决定下一步该做什么。**

---

## 四、智能体 (Agent) 的决策模式 (4.png)

* **定义**：自己决定下一步怎么做，而非照搬死板流程（围绕目标自己规划路线）。
* **工作模式**：`输入任务` -> `[ Agent 黑盒决策 ]` -> `输出结果`。
* **同样任务下的 Agent 思考过程示例**：
  1. *思考*：要发上次会议纪要，那上次会议是什么时候？
  2. *尝试*：去查系统日历 -> 反馈：找不到日历项。
  3. *决策（不放弃，寻找替代方案）*：尝试改去腾讯会议或本地会议文件夹检索最近的会议记录。
  4. *执行与总结*：拿到会议记录文件，调用大模型进行纪要摘要。
  5. *遇到缺失信息*：发现用户没提供收件人邮箱，主动暂停并反问用户：“请问纪要需要发送到哪个邮箱？”。
* **核心特征**：
  * **自主思考**：根据目标和遇到的实际阻碍，动态调整行为路径。
  * **工具选择**：自己决定在什么时候、使用什么工具（如日历、搜索、反问）。

---

## 五、完整 Agent 的五个常见组成部分 (5.png)

理解 Agent 可以类比为“招聘一个数字员工（24小时待命）”：

1. **LLM 大脑 (LLM Brain)**：
   * **作用**：理解人类语言、分析拆解任务、制定策略计划。
   * **定位**：核心决策中枢。
2. **Prompt 岗位说明书**：
   * **作用**：规定 Agent 的职责范围、行为边界和回复风格风格人设。
   * **定位**：约束与人设指导。
3. **Memory 记忆系统**：
   * **作用**：记住对话上下文、用户偏好和历史执行经验。
   * **定位**：上下文及经验库。
4. **Knowledge 专业知识库**：
   * **作用**：提供专业领域的企业文档、产品资料或操作规章，作为事实依据。
   * **定位**：背景参考资料。
5. **Tools 手脚/工具**：
   * **作用**：执行具体动作（如发邮件、查网页、做表格、下订单），实现与外部物理系统的交互。
   * **定位**：执行实体。

---

## 六、Tools（工具）的本质与边界 (6.png)

* **本质**：Tools 是给 Agent 戴上的“手和脚”，是将 AI 从“会聊天”提升到“能干活”的关键一步。
* **形式**：这里的工具不是物理扳手，而是电脑、手机或服务器中的 API 操作权限（读写权限）。
* **对比**：
  * **没有 Tools**：只能文字聊天，无法查询日历、无法发送邮件、无法操作本地/外部软件。
  * **引入 Tools**：能够自主查询网页/日历、读取会议文档、生成 Excel/PPT、发送邮件或下单订票。
* **边界与安全**：Agent 获得的权限（API）越多，能做的事就越广；但相应地，越需要设置清晰的边界与安全限制（防止误操作或恶意执行）。

---

## 七、真 Agent 的关键：ReAct 循环迭代机制 (7.png)

* **核心概念**：**ReAct (Reasoning + Acting)** —— 思考与行动的有机循环（注意：不是前端前端 React 框架）。
* **Agent Loop 环形流程**：
  $$\text{思考 (Reasoning)} \longrightarrow \text{行动 (Acting)} \longrightarrow \text{观察结果 (Observation)} \longrightarrow \text{自我检查 (Self-Check)} \longrightarrow \text{迭代/输出} \longrightarrow \text{继续思考}$$
* **任务示例**：“帮我做一份竞品分析”
  * **第一步**：自主打开网页，检索并寻找合适的竞品；
  * **第二步**：收集整理数据，画出对比图表；
  * **第三步**：自我检查结果，评估是否符合任务目标；
  * **第四步**（不满意）：针对不足点，继续修改和补充数据；
  * **第五步**（满意）：最终输出完善的结果报表。
* **核心结论**：**会循环迭代，才是真 Agent**。Agent 不是像传统的程序或普通 LLM 那样“单次运行、一次交差”，而是具备“自我评估 -> 发现缺失 -> 迭代修正”的闭环迭代能力。
